Resolving a revert conflict — the recipe

  1. Open the conflicted file and find the <<<<<<< … >>>>>>> markers.
  2. Choose the correct code and delete the markers and the parts you don't want.
  3. Stage the fixed file with git add <file>.
  4. Finish the revert with git revert --continue.
git add <file>

Marks a conflicted file as resolved, once you've edited it to the version you want.

git revert --continue

Tells Git you're done resolving — it finishes the revert and creates the revert commit.

git revert --abort

Changed your mind? Cancels the whole revert and puts everything back exactly as it was before you started.

git status

Lost? Run this any time — it shows which files are still conflicted and what to do next.

✅ The big idea: revert is safe — it never silently throws work away. When two changes collide on the same place, Git simply stops and asks you to decide. Don't worry about mastering conflicts here; there's a separate lesson dedicated to resolving them. The takeaway: revert is safe, but sometimes it needs your help.
