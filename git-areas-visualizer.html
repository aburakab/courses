Good to know: what does HEAD~1 mean?

Several undo commands point at a commit using HEAD~1. It looks cryptic, but it's simple once you break it apart.

HEAD where you are now

HEAD is a pointer to your current commit — normally the newest commit on the branch you're on (the very top of the history).

~1 step back in time

The ~ (tilde) followed by a number means “go back that many commits” by following each commit's parent. So HEAD~1 = the commit right before the current one.

ReferencePoints to
HEADcurrent commit (the latest)
HEAD~11 commit back — the parent
HEAD~22 commits back — the grandparent
HEAD~33 commits back, and so on…
Example — with this history:
c3d  Add feature   ← HEAD     (current)
a1b  Init          ← HEAD~1   (one before)

git reset HEAD~1 moves your branch back to a1b, so commit c3d is no longer the current commit. What happens to c3d's changes depends on the flag — --soft keeps them staged, --mixed (the default) keeps them in your working dir, and --hard throws them away. Step through the Undo flow above to see each one.

💡 HEAD~ with no number is the same as HEAD~1. You'll also see HEAD^ — in normal (non-merge) history it means the same thing. They only differ at merge commits, which have two parents; until you hit those, just read HEAD~1 as “the previous commit.”
